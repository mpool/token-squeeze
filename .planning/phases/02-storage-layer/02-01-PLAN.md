<objective>
Refactor IndexStore to use constructor-injected cache directory, add CACHEDIR.TAG and .gitignore marker creation, and update all callers and tests.

Purpose: IndexStore currently passes cacheDir to every method call. Moving it to the constructor simplifies the entire call chain and makes the API impossible to misuse (can't accidentally mix cache directories). Adding cache markers ensures backup tools skip the cache and git ignores it automatically.

Output: IndexStore with constructor(cacheDir), all ~60 call sites updated, cache markers written on Save(), tests using constructor injection.
</objective>
