  <name>Task 1: Rewrite IndexStore to use cacheDir parameter</name>
  <files>src/TokenSqueeze/Storage/IndexStore.cs</files>
  <action>
Rewrite IndexStore to thread `cacheDir` through all methods. This is the largest change in Phase 1.

**Constructor:**
- Remove `StoragePaths.EnsureRootExists()` call from constructor. The empty constructor can remain or be removed entirely. Directory creation happens in `Save()` only (aligns with STOR-04 in Phase 2, but we prep for it now).

**Every method** that currently takes `string projectName` changes to `string cacheDir`:
- `Save(CodeIndex index, string cacheDir)` -- add cacheDir parameter
- `LoadManifest(string cacheDir)` -- was `LoadManifest(string projectName)`
- `LoadFileSymbols(string cacheDir, string filePath)` and the overload with Manifest
- `LoadAllSymbols(string cacheDir)`
- `Load(string cacheDir)`
- `DeleteFileFragment(string cacheDir, string storageKey)`
- `SaveFileFragment(string cacheDir, string storageKey, FileSymbolData fragment)`
- `SaveManifest(string cacheDir, Manifest manifest)`
- `RebuildSearchIndex(string cacheDir, Manifest manifest)`
- `UpdateSearchIndex(string cacheDir, Manifest manifest, HashSet<string> affectedFiles)`

**Inside each method**, replace:
- `StoragePaths.GetProjectDir(projectName)` => just use `cacheDir` directly
- `StoragePaths.GetManifestPath(projectName)` => `StoragePaths.GetManifestPath(cacheDir)`
- `StoragePaths.GetFilesDir(projectName)` => `StoragePaths.GetFilesDir(cacheDir)`
- `StoragePaths.GetFileFragmentPath(projectName, key)` => `StoragePaths.GetFileFragmentPath(cacheDir, key)`
- `StoragePaths.GetSearchIndexPath(projectName)` => `StoragePaths.GetSearchIndexPath(cacheDir)`

**Security boundary (SEC-01):**
- Old: `PathValidator.ValidateWithinRoot(projectDir, StoragePaths.RootDir)` -- validated project dir within global root
- New: `PathValidator.ValidateWithinRoot(fragmentPath, cacheDir)` -- validate fragment paths within cache dir
- The cacheDir itself does NOT need validation against a parent (it IS the root boundary)
- Fragment paths and file paths DO get validated against cacheDir

**Save method specifics:**
- `Directory.CreateDirectory(cacheDir)` and `Directory.CreateDirectory(filesDir)` -- create on save
- Remove `index.ProjectName` references -- use `index.SourcePath` where needed
- Set `FormatVersion = 3` (bumped from 2, per research recommendation -- format is incompatible)
- Remove `Manifest { ProjectName = ... }` from construction
- Remove `UpdateCatalog()` call entirely
- Remove legacy file cleanup block (lines 99-106 referencing `GetLegacyIndexPath` and `GetMetadataPath`)

**Load method specifics:**
- Remove entire legacy fallback block (the `GetLegacyIndexPath` branch, ~lines 215-221)
- When constructing CodeIndex from manifest, do NOT set ProjectName (it no longer exists)

**DELETE entirely:**
- `LoadMetadata(string projectName)` method -- ProjectMetadata is deleted
- `IsLegacyFormat(string projectName)` method -- no migration
- `GetLegacySourcePath(string projectName)` method -- no migration
- `LoadManifestHeader(string projectName)` method -- ManifestHeader is deleted
- `ListProjects()` method -- no global storage
- `Delete(string projectName)` method -- purge command deleted in Phase 3 (but method can stay temporarily if needed for compilation; prefer removing it)
- `UpdateCatalog()` private method -- no catalog
- `LoadCatalogJson()` method -- no catalog

**KEEP unchanged:**
- `AtomicWrite<T>` and `AtomicWriteRaw` -- generic utilities, no project concept
- `ProjectSearchSymbols` -- pure projection, no project concept
